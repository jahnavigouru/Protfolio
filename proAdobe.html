    • Internship

    Part of Adobe campaign's UCO team. UCO, the "Unified Campaign Operations" is a self-servicing platform that unifies all the operational-oriented tools required for the Adobe Campaign. The first 3-weeks of the 6-month long internship were the learning phase, which involved implementing key-value as a service using golang, react, MongoDB, and docker for dockerizing all the services.

    This internship revolved around exploring, designing, and developing internal tools for the UCO to improve the consistency and reliability of campaign infrastructure and speed up the value delivery of rollouts on Adobe campaign.

    Features

    • Scheduling Assistant
    • Very very very very long link

    Theme 1: Speed Up Value Delivery

    Adobe campaign requires numerous continuous deployments to avoid these rollouts getting scheduled in the same time frame and conflicting with any pre-existing rollouts, the UCO team needed a way to provide PgMs more visibility into the rollout schedule.

    At present, it was done by some intermediary which isn't feasible given the wide range of rollout adoption.

    Business value:
  • Help PgMs plan better rollouts.
  • Better UX in rollout planning

    • Solution

    Scheduling Assistant
  • This provides one single view with all the scheduled releases of the products (ACC / ACS ) in the form of a calendar.
  • This information helps in effectively scheduling releases.
  • Color coding is used to indicate if the slots are exceeding the product's execution capacity so that the user stays warned while scheduling a new release.
  • Provision to show change restricted slots to align with Adobe wide change restrictions as well as internal UCO specific releases.

    • Feature Implementation Process

    After understanding UCO's complex architecture, the problem statement, and the impact of scheduling assistant as a feature. I started to implement the feature.
